# 产品需求文档 (PRD) v1.0


## 一、产品概述

- 产品目标：为工作室提供一套集学员管理、作业打卡、自动生成反馈图、课时统计于一体的轻量化管理工具。
- 核心价值：减少老师在微信重复沟通的时间，通过一键生成反馈图提升品牌专业形象。
- 技术选型：Taro 4.0.9 + React 18 (支持 H5、微信小程序等多端)，后端可配合云开发或轻量化本地数据库。

## 二、核心功能模块划分

### 1. 学员管理系统 (User CRM)

不仅是信息录入，更是后续消课统计的基础。

- 状态管理：分为"在读"、"暂停（请假）"、"结业"三种状态。
- 属性字段：姓名、昵称（用于反馈图）、性别、生日、剩余课时、收费标准（元/次）、备注、头像。
- 智能头像：若无上传头像，系统根据姓名最后一个字自动生成：
  - 逻辑：随机背景色（莫兰迪色系）+ 白色文字居中。

### 2. 库管理 (Library)

- 作业库 (Task Bank)：预设常用的练习项（如：拼音读拼、20以内加减法、看图说话等）。
  - 支持首字母拼音快捷搜索（如输入 PY 匹配 拼音）。

### 3. 费用与消课统计 (Finance & Stats)

- 扣费逻辑：点击"完成今日登记"或"生成反馈"时触发扣费（-1课时）。
- 阈值预警：当剩余课时 ≤ 3次时，在首页或统计页给予橙色高亮提示。
- 报表：月度/周度消课总数趋势图，清晰掌握工作室运营负荷。

### 4. 反馈模板管理
- 反馈模版管理：预设反馈图的视觉风格（简约、可爱、专业）。

## 三、核心业务流程：每日反馈链路

这是该应用最频繁使用的场景，设计目标是："30秒内完成一个学生的反馈"。

### Step 1：首页 - 考勤打卡 (Attendance)

- 视图：采用 GridView（宫格视图） 展示所有"在读"学员。
- 交互：点击头像多选（或一键全选），点击底部大按钮 「完成今日登记」。
- 系统动作：批量创建今日待反馈记录，并扣除相应课时（注意：可能有学员忘记续课，支持展示负数）。

### Step 2：反馈中心 - 任务编辑 (Feedback Detail)

- 顶部导航：横向滚动展示今日已签到的学生头像。
- 记忆功能：点击学生头像，下方内容区自动载入该学生昨日的作业项（减少输入量）。
- 任务编辑：
  - 自动展示该学员昨天的作业
  - 点击 + 号弹出作业库，支持搜索，添加到当前反馈里
  - 支持手动修改每项状态（已完成、需加强、未带书）
  - 支持侧滑删除任务，点击切换任务状态
- 状态反馈：点击"完成反馈"按钮，系统弹出 Loading 状态，成功后展示"已存至本地"动画。

### Step 3：图片生成与分享 (Image Generation)

- 生成逻辑：使用Taro的Canvas API或相关组件库（如taro-canvas-painter）来处理反馈图生成，确保在不同设备上生成的反馈图比例一致。
- 反馈图内容包含：
  - 头部：工作室 Logo、今日日期
  - 主体：学生头像、昵称、今日学习进度列表
  - 底部：老师寄语（支持快捷语录）、剩余课时提醒

## 四、页面设计原型思路 (UX Design)

| 页面名称 | 视觉核心 | 关键交互 |
| --- | --- | --- |
| 首页 (Home) | 活跃学生卡片阵列 | 长按头像进入编辑，单击多选 |
| 反馈页 (Process) | 分屏设计：上选人，下写内容 | 搜索框支持拼音简码，快速添加作业 |
| 统计页 (Dashboard) | 环形图 + 预警列表 | 点击预警用户，可直接拨打电话或复制微信名 |
| 设置页 (Settings) | 列表式功能菜单 | 作业库管理、反馈图模版更换、数据导出备份 |
