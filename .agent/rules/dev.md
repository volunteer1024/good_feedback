# 🏗️ 学成反馈系统 - 开发指南 (Taro)

> **项目目标**：为工作室老师提供高效的学员打卡、学习反馈记录及记录图生成工具。
>
> **当前状态**：数据服务暂基于本地缓存 (`Taro.setStorage`)，后续迁移至微信小程序云开发。

---

## 🛠️ 技术栈 (Tech Stack)

| 维度         | 技术选型                 | 说明                                           |
| :----------- | :----------------------- | :--------------------------------------------- |
| **前端框架** | Taro (React)             | 跨端适配主流小程序及 H5                        |
| **编程语言** | TypeScript               | 核心开发语言（项目当前处于 JS 向 TS 迁移阶段） |
| **后端方案** | 微信小程序云开发         | 初期本地缓存，后期云函数 + 云数据库            |

## 📂 目录分层逻辑 (Architecture Layering)

严格遵守以下分层规范，禁止跨层直接调用，以确保代码的可维护性和可测试性。

| 目录              | 职责             | 说明                                               |
| :---------------- | :--------------- | :------------------------------------------------- |
| `src/pages/`      | **页面视图层**   | 负责 UI 渲染和调用 Hooks。**禁止直接调用云函数**。 |
| `src/components/` | **公共 UI 组件** | 保持组件纯粹性，通过 Props 进行数据传递。          |
| `src/services/`   | **核心抽象层**   | **非常重要**：封装所有后端请求和本地存储逻辑。     |
| `src/hooks/`      | **业务逻辑层**   | 处理复杂的页面逻辑、表单状态或组合数据流。         |
| `src/utils/`      | **工具类**       | 提供日期格式化、跨端兼容判断等通用方法。           |
| `src/constants/`  | **常量定义**     | 存放云环境 ID、业务配置、错误码等。                |
| `cloudfunctions/` | **后端代码**     | 每个云函数应作为独立、单一职责的模块。             |

## 规范与准则 (Standards)

### 1. 编码准则

- **跨端适配**:
  - 使用 `process.env.TARO_ENV` 处理平台差异。
  - 样式优先使用 `rpx` 或 `rem`。
- **后端调用**:
  - 所有请求返回统一格式：`{ code: number, data: any, msg: string }`。
  - 必须使用 TypeScript 定义接口的输入 (Payload) 与输出 (Response)。
  - **标准封装示例**:
    ```typescript
    // src/services/base.ts
    export const callCloud = async <T>(name: string, data?: any): Promise<T> => {
      const res = await Taro.cloud.callFunction({ name, data });
      return res.result as T;
    };
    ```
- **UI 组件**:
  - 所有 Icon 组件需使用 Taro 框架原生的 Icon 库。
- **状态管理**:
  - 优先使用 React Context 或 Zustand 管理全局状态（如用户信息）。
  - 使用 TanStack Query 处理服务器缓存，避免手动管理大量的 `loading` 变量。

### 2. 云函数开发规范

- **单一职责**: 一个云函数只做一件事（或通过 `tcb-router` 管理子路由）。
- **环境隔离**: 严格区分 `dev` 和 `prod` 环境 ID。
- **冷启动优化**: 减少云函数包体积，尽量复用公共层 (Layer)。

### 3. Cursor 交互建议

在使用 AI 协作编程时，请遵循：

1.  **架构引导**: 要求 AI 先生成 `service` 和 `type` 定义，再编写 `hook`，最后实现 `page`。
2.  **类型优先**: 始终优先定义 Interface。
3.  **健壮性**: 明确要求云函数调用包含 `try-catch` 及统一的错误提示。

## 💡 核心功能实现逻辑

### 🎨 反馈图生成 (Canvas)

- **方案**: 使用 Taro Canvas API 或 `taro-canvas-painter`。
- **要素**: 头部 (Logo/日期/星期)、主体 (学生个性化头像/昵称/学习进度列表)。
- **一致性**: 确保不同分辨率设备上生成的图片比例一致。

### 👤 智能头像系统

- **逻辑**: 若无照片头像，根据姓名最后一位自动生成。
- **视觉**: 随机莫兰迪色系背景 + 白色文字居中。
- **实现**: CSS Flex 布局 + `substring(name.length - 1)`。

### 💾 离线与同步

- **本地优先**: 利用 `Taro.setStorage` 缓存学生名单。
- **背景**: 解决工作室地下室信号差的问题。
- **同步**: 记录先存本地，待环境网络恢复后手动/自动同步至云端。

### 📊 业务流程

- **考勤打卡**: GridView 宫格展示学员 -> 点击头像多选 -> 点击「完成今日登记」。
- **消课逻辑**: 点击签到或生成反馈时，触发 `课时 - 1`。支持负数课时预警。
- **反馈中心**: 顶部横向滚动学生列表 -> 记忆功能（载入昨日内容）-> 作业库搜索添加。
